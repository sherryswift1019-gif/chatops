## 前置：可行性评估

在头脑风暴之前，先做了产品可行性评估，结论：**可行，但需要调整策略**。

### 有利条件

- 团队已全面 AI 编程，容错度高
- Bug 量每月几十个，成本可控
- 单测覆盖 70% 且在提升，自动验证可行
- 70% 成功率即可 + 人机协同

### 成功率修正

原文档预期端到端成功率偏乐观，修正后：
- AI 分析 + 人确认(85%) × AI 修复(75%) = 64%
- 加上知识库命中复用，整体有效率可达 70%+ → 达标

### 已确认的策略调整（已更新到需求文档）

1. **分级路由流程** — L1/L2 快通道、L3 保留审批、L4 纯辅助
2. **分级判断机制** — AI 自动分级，分错代价可控（3 次重试后自动降级）
3. **失败路径** — 独立 fix 分支，不污染主分支
4. **AI 摘要维护** — 纳入修复 Agent 工作流，随代码同步更新
5. **平台统一** — 直接 GitLab Issue，移除禅道

## 阶段 2：找规律（思维导图聚类）

### 模式 A：产品进化路径

```
效率工具 → 安全产品 → 全角色 AI 同事 → 无人值守代码工厂
```

不是并行，是进化链。每一步的市场、买单人、定价都不同。

### 模式 B：壁垒飞轮

```
私有化进场 → 积累知识 → AI 更准 → 客户更信任 → 更多数据 → 更强壁垒
```

技术没壁垒（"大自然的搬运工"），壁垒在 B+C：编排层 + 垂直领域数据。

### 模式 C：多客户画像

同一个入口（钉钉 @机器人），背后不同处理链路：
- 研发 → Bug 分析修复流程
- 交付/售后 → 知识库查询 + 问题分析
- 安全合规 → CVE 扫描 + 代码审计报告

### 模式 D：数据洞察层

不是统计 Bug 数量，是追溯 Bug 从哪来：

```
需求描述 → prompt → AI 摘要 → AI 理解 → AI 写代码
```

每个 Bug 都是信息链条的诊断机会。现在代码都是 AI 写的：
- 纯语法错误 → 模型/代码规范问题
- 业务逻辑错误 → prompt/摘要没说清业务规则
- 需求理解偏差 → PRD 描述模糊
- 边界条件遗漏 → 测试用例模板缺失
- 跨模块冲突 → 架构约束没文档化

**核心洞察（Hanff）**：Bug 的 owner 不一定是 AI 编码能力差，可能是提示词问题或需求没说清楚。数据洞察层的价值是驱动整个知识体系进化。

## 阶段 3：六顶思考帽

### 白帽（事实）

已知事实：全面 AI 编程、Bug 量不大、单测 70%、钉钉+GitLab 已跑通、私有化是硬需求、编排层无技术壁垒。

缺失事实（待验证）：AI 分级准确率、L1/L2 占比、竞品动态。Hanff 判断：都可以在实践中验证，当前阶段不阻塞。

### 红帽（直觉）

- **兴奋**：自动化、高效解决问题
- **担心**：成效（AI 真的能达到预期吗？）

### 黄帽（价值）

三板斧就够 MVP 站住脚：分析报告 + 知识库秒回 + L1 自动修复。

### 黑帽（风险）+ 绿帽（化解）

| 风险 | 化解方案 | 本质 |
|------|---------|------|
| AI 分析质量不稳定 → 信任崩塌 | 置信度标签，AI 自标可信度 | 预期管理 |
| 流程太长没人走完 | 环节可选，支持最小 MVP | 渐进增强 |
| 自动修复反而增加工作量 | 分析阶段就介入引导生成方案，再交 Agent 执行 | 方案先行 |
| 知识库冷启动 | AI 主动扫描初始化，能补的自己补，不能的问人 | 主动填充 |

### 蓝帽（全局）

产品定位从"研发效率工具"演化为三层：
1. **第一层**：研发效率工具（MVP）
2. **第二层**：全产品知识大脑（全角色 AI 同事）
3. **第三层**：越用越值钱的平台（数据壁垒 + 生态锁定）

## 阶段 4：落地（决策树）

### 三个里程碑

**里程碑 1：分析闭环** — 知识库初始化、置信度标签、方案先行、自动分级、知识库命中秒回
- 交付标准：分析报告 + 置信度 + Bug 级别 + 修复方案
- 独立可用，交付/售后人员直接能用

**里程碑 2：修复闭环** — GitLab Webhook、修复 Agent（L1→L2）、失败降级、AI 摘要同步、AI Review
- 交付标准：L1 全自动到 MR，L2 半自动
- 核心差异化阶段

**里程碑 3：进化闭环** — Bug 根因归因、知识库自动沉淀、价值量化、L3 流程、多模型可插拔
- 交付标准：Bug 根因可追溯、知识库自动增长、AI 成效可量化
- 飞轮转起来

### 决策原则

- 每个里程碑完成后可停下评估，不用一口气做完
- 里程碑 1 是基础，必须先做扎实
- 里程碑 2 是差异化，做完才算真正的产品
- 里程碑 3 是壁垒，越用越值钱
